# Getting Started with NVIDIA NemoClaw on PapayaClaw

At **GTC 2026**, NVIDIA unveiled **NemoClaw** — an open-source stack that brings enterprise-grade security, privacy, and local inference to [OpenClaw](https://openclaw.com) AI agents. In this guide, we'll walk you through what NemoClaw is, why it matters, and how to get it running on **PapayaClaw** step by step.

## What is NemoClaw?

NemoClaw is NVIDIA's official plugin for the OpenClaw agent platform. It installs **NVIDIA Nemotron models** and the **OpenShell runtime** with a single command, wrapping your AI agents ("claws") in multiple layers of security and privacy controls.

Think of it this way: if OpenClaw is the operating system for personal AI agents, **NemoClaw is the security and performance layer** that makes those agents trustworthy, private, and production-ready.

### Key Features

- **Local Inference**: Run NVIDIA Nemotron models locally — no token costs, no data leaving your machine.
- **Sandboxed Execution**: Every agent runs inside an isolated OpenShell container with filesystem, network, and process-level restrictions.
- **Multi-Layer Security**: Combines Landlock, seccomp, and network namespace isolation (netns) for defense in depth.
- **Privacy Router**: When you need cloud models, NemoClaw routes requests through a controlled inference gateway — no direct API calls from inside the sandbox.
- **Hot-Reloadable Policies**: Update security and network rules at runtime without restarting your sandbox.
- **Policy-as-Code**: Declarative, auditable security configurations you can version control.

### Architecture Overview

NemoClaw is composed of four layers:

1. **Plugin Layer** — A TypeScript CLI for sandbox lifecycle management (`launch`, `connect`, `status`, `logs`).
2. **Blueprint System** — Versioned Python orchestration artifacts that configure sandbox creation, security policies, and inference providers.
3. **Sandbox Environment** — Isolated OpenShell containers running OpenClaw with policy enforcement.
4. **Inference Gateway** — Routes all model API calls through NVIDIA's cloud infrastructure or local Nemotron models.

## Hardware Requirements

This is where it gets important. NemoClaw has specific minimum requirements:

| Resource | Minimum | Recommended |
|----------|---------|-------------|
| **CPU** | 4 vCPU | 4+ vCPU |
| **RAM** | 8 GB | 16 GB |
| **Storage** | 20 GB free | 40 GB free |

The sandbox image alone is approximately **2.4 GB compressed**. During deployment, Docker, k3s, and the OpenShell gateway run alongside the export pipeline, which buffers decompressed layers in memory. Systems with less than 8 GB RAM risk triggering the **OOM killer**.

## Step-by-Step Installation Guide

### Step 1: Subscribe to the Pro Plan

Head to [PapayaClaw Pricing](/pricing) and subscribe to the **Pro plan** ($17.90/month). Once your instance is provisioned, connect via SSH or Telegram.

### Step 2: Connect to Your Instance

You can connect via SSH or directly from the PapayaClaw dashboard. In the dashboard, navigate to the **Root Terminal** and click the **ROOT@ CONNECT** button:

![Click ROOT@ CONNECT to open the shell](/blog/getting-started-with-nemoclaw/nemoclaw_1.png "PapayaClaw dashboard — click ROOT@ CONNECT to access your instance")

Once connected, you'll see the terminal with the OpenClaw status and system information:

![Connected to the shell with OpenClaw status](/blog/getting-started-with-nemoclaw/nemoclaw_2.png "Terminal connected — OpenClaw status overview")

### Step 3: Ensure Docker is Running

```bash
docker --version
# Should output Docker version 24.x or later

sudo systemctl status docker
# Ensure it shows "active (running)"
```

### Step 4: Install NemoClaw

Run the single-command installer:

```bash
curl -fsSL https://nvidia.com/nemoclaw.sh | bash
```

This script will:
1. Check and install Node.js if needed.
2. Launch an interactive onboarding wizard.
3. Create your sandbox environment.
4. Configure inference settings.
5. Apply default security policies.

### Step 5: Complete the Onboarding Wizard

The wizard will ask you to:

1. **Name your sandbox** — Choose something descriptive like `my-assistant`.
2. **Select an inference provider** — Choose between NVIDIA Cloud API (recommended for Pro plan) or local models.
3. **Enter your API key** — Get one free at [build.nvidia.com](https://build.nvidia.com).
4. **Configure security policies** — The defaults (Landlock + seccomp + netns) are recommended.

After completion, you'll see a summary like this:

```
  ┌──────────────────────────────────────────────┐
  │   NemoClaw Deployment Summary                │
  │                                              │
  │   Sandbox:    my-assistant                   │
  │   Security:   Landlock + seccomp + netns     │
  │   Model:      nvidia/nemotron-3-super-120b   │
  │   Provider:   NVIDIA Cloud API               │
  │   Status:     Running ✓                      │
  └──────────────────────────────────────────────┘
```

### Step 6: Connect to Your Sandbox

```bash
nemoclaw my-assistant connect
```

This drops you into the isolated sandbox shell:

```
sandbox@my-assistant:~$
```

### Step 7: Start Using Your Agent

**Interactive chat (TUI):**

```bash
openclaw tui
```

**Single command:**

```bash
openclaw agent --agent main --local -m "Hello, help me write a Python script" --session-id test
```

**Check status anytime:**

```bash
nemoclaw my-assistant status
```

## Security Deep Dive

NemoClaw's security operates on four layers:

1. **Network Layer** — Blocks all unauthorized outbound traffic. Any unapproved connection attempt surfaces for operator approval.
2. **Filesystem Layer** — Restricts access to `/sandbox` and `/tmp` only. The agent cannot read or modify system files.
3. **Process Layer** — Prevents privilege escalation and blocks restricted syscalls via seccomp profiles.
4. **Inference Layer** — Intercepts all model API calls and routes them through controlled backends, preventing data exfiltration via prompt injection.
