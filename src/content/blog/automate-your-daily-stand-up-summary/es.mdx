## El Problema con las Reuniones Diarias

Las reuniones diarias deberían ser simples. 15 minutos, cada uno dice qué hizo y qué va a hacer. Pero en la práctica?

- Estás escuchando cómo los desarrolladores describen su proceso de pruebas durante 5 minutos
- Alguien se va por las ramas sobre cambios en infraestructura
- El desarrollador senior explica cómo refactorizaron un servicio que ni siquiera usas
- Al terminar, sigues sin saber quién tiene bloqueos hoy

Para cuando termina la reunión, tienes páginas de notas pero aún no puedes responder: ¿quién necesita ayuda hoy?

### 2. Escribe la Lógica del Resumen

El agente extrae información clave de la reunión:

```python
def resumir_reunion(transcripcion):
    """Extrae puntos clave de la reunión diaria."""
    
    # Encontrar bloqueos
    bloqueos = extraer_secciones(
        transcripcion,
        palabras_clave=["bloqueado", "bloqueado por", "esperando", "dependencia"]
    )
    
    # Encontrar actualizaciones de progreso
    progreso = extraer_secciones(
        transcripcion,
        palabras_clave=["completado", "terminado", "hecho", "desplegado"]
    )
    
    # Encontrar acciones
    acciones = extraer_acciones(transcripcion)
    
    return {
        "bloqueos": bloqueos,
        "progreso": progreso,
        "acciones": acciones,
        "resumen": generar_resumen(bloqueos, progreso, acciones)
    }
```

### 3. Envía el Resumen a Slack

Configura el agente para publicar el resumen en el canal de Slack de tu equipo:

```python
def publicar_en_slack(resumen):
    """Envía resumen de reunión a Slack."""
    
    mensaje = (
        "# Resumen Diario - Hoy\n\n"
        f"🔴 **Bloqueos:** {len(resumen['bloqueos'])}\n\n"
        "## Actualizaciones de Progreso\n"
        f"{resumen['progreso']}\n\n"
        "## Acciones\n"
        f"{resumen['acciones']}\n\n"
        "---\n"
        "📋 Transcripción completa disponible en PapayaClaw"
    )
    
    slack_client.chat_postMessage(
        channel="#reuniones",
        text=mensaje,
        mrkdwn=True
    )
```
